# 0.2 release notes

## New features

### Full MO2 profile import

**File → Import MO2 Profile...** replaces the old single-file import.
Point at the MO2 instance folder (where `ModOrganizer.ini` lives);
profiles are listed; mods are matched against the actual folder in
MO2's `mods/` directory and linked directly when present (status:
installed). MO2's `@ByteArray(...)` path encoding and Windows
backslashes are handled automatically. If the selected profile has a
`plugins.txt`, an optional checkbox imports the load order too.

The old **File → Import from MO2 (modlist.txt)...** entry is still
available as a fallback.

### Per-mod changelog viewer

Right-clicking any mod with a Nexus URL shows **View Changelog...**.
Opens a non-modal dialog that fetches the full Nexus changelog and
shows it version by version. You can open several at once for
different mods. Requires a Nexus API key.

### Wabbajack modlist import

**File → Import Wabbajack Modlist...** imports a `.wabbajack` file.
Extracts the embedded JSON, reads the `Archives` array, and creates
not-installed placeholders for every mod sourced from Nexus, pre-filled
with name, URL and file size. Drag-and-drop also works: drop a
`.wabbajack` file onto the window.

### Drag-and-drop import for MO2 / Wabbajack files

In addition to the menu entries, you can drop these onto the mod list:

- `.wabbajack`: starts the Wabbajack import flow.
- `modlist.txt`: contents are peeked to distinguish MO2 from
  Nerevarine format, then the right importer is launched.

Confirmation dialog before replacing the current list.

### Foreign modlist path remapping

When a Nerevarine `modlist.txt` is imported from another machine, the
absolute mod paths won't exist locally. The importer detects this and
lets you pick your local mods folder; every path is rewritten before
the list is populated. No dialog when paths already exist locally.

### Search on Nexus for mods without a known URL

Right-clicking a not-installed placeholder with no Nexus URL shows
**Search on Nexus...** at the top of the context menu. Opens the
browser to a Nexus keyword search for the mod name. For mods with a
stored non-Nexus URL (e.g. a GitHub release), **Open download page**
appears instead.

### MO2 import resolves Nexus URLs from `meta.ini`

Both MO2 import paths now read each mod's `meta.ini` to pre-fill the
Nexus URL. MO2 stores Nexus mod IDs in
`<modsDir>/<ModName>/meta.ini` under `[General]/modid`. When present
(and `repository=Nexus` or absent), the importer constructs the full
URL so the one-click Nexus download flow is immediately available.

### Case-variant asset detection in Modlist Inspector

**Inspect OpenMW Setup** now flags files within the same data folder
whose relative paths differ only by case (e.g. `Scripts/Player.lua`
and `Scripts/player.lua`). On Linux these are two distinct files;
OpenMW's VFS loads both and the last `data=` wins, silently shadowing
the other.

Pure detection logic in `src/asset_collisions.cpp` /
`include/asset_collisions.h`; 18 unit tests in
`tests/test_asset_collisions.cpp`. Loose files only (not BSA), single
data root only.

### AppImage now bundles Qt

The Linux release ships as a self-contained AppImage. Previously a
raw binary against the developer's system Qt (6.11), which broke on
distros with older Qt -- including Steam Deck (locked to Qt 6.8).
Bundles its own Qt, runs on any x86_64 Linux with glibc 2.17+
regardless of system Qt.

Building from source still uses your system Qt 6.x (6.4+).

## Bug fixes

### Wabbajack import froze the UI

Large Wabbajack files (some Skyrim lists exceed 50 MB of JSON) caused
the UI to freeze during 7z extract + JSON parse. Now runs on a
background thread via `QtConcurrent::run`, with a window-modal
indeterminate progress dialog.

### `.rar` extraction failed on `unrar-free`

`.rar` archives previously used `unrar` only. Many distros ship
`unrar-free`, which exits with code 3 on RAR5 and solid archives.
Falls back to `7z` (with `p7zip-full`'s RAR codec) when `unrar` fails
or is missing.

If both fail, the error dialog now names the package to install
(`pacman -S unrar`, `apt install unrar` / `apt install p7zip-full`).

### Nexus 403 error message was wrong

Nexus 403 means the mod is hidden / removed / behind adult-content
gating, not a bad API key. The dialog now says so.

### Launch warning fired for script-only mods

The pre-launch sanity check flagged any enabled, installed mod with no
recognised plugin or asset folder as a potential empty install -- a
false alarm for `.lua`-only mods etc. Now suppresses the warning if
the folder contains any files at all.

### FOMOD wizard windows blocked each other

Queuing several FOMOD installs at once forced the user to complete each
modal wizard in order. Wizards are now non-modal independent top-level
windows; you can switch between them and the rest of the UI freely.
Each wizard drives its own post-install logic via an internal callback.

### OpenMW-specific dependency warnings suppressed

The pre-launch missing-dependency check warned about MGE XE, MWSE, and
"Mod Organizer to OpenMW" requirements -- always false positives for
Nerevarine users. These labels are now silently filtered.

### 7z exit codes explained clearly

The error dialog now distinguishes:

| Code | Meaning |
|------|---------|
| 1 | Warning: some files unreadable (partial extract succeeded) |
| 2 | Fatal: archive corrupt or unsupported format |
| 255 | User abort or out-of-memory |

### Non-Nexus mods had no install option

Mods added without a Nexus URL (e.g. imported from a Wabbajack list
with a GitHub source) had no install path. The context menu now shows
**Install from local archive...** and, when a non-Nexus URL is on
file, **Open download page**.

This also fixed shader-only mods (no `.esp`/`.esm`, no recognised
asset folders) that were not detected as installed. The startup
`repairEmptyModPaths` self-heal now counts a mod as valid if its
folder has any files at all.

## Improvements

### FOMOD installer prefers English

When a FOMOD presents a step that looks like a language selector, the
wizard now scores `English` / `EN` options above the FOMOD-declared
`Recommended` default:

1. Option name contains "OpenMW" (existing).
2. Option name contains "English" or is exactly "EN" (new).
3. The FOMOD `Recommended` / `Required` plugin (fallback).

Stops Russian / other languages being picked when the FOMOD doesn't
mark English as `Recommended`.

### Wabbajack import filters tools and dedups

- Tool filtering: archives whose name matches a known engine-tool
  prefix (Mod Organizer, MWSE, MGE XE, xEdit/TES3Edit, LOOT, Wrye
  Mash, openmw-csh, mlox) are skipped.
- Duplicate Nexus mod IDs are merged: shortest display name wins,
  largest recorded file size wins.

### Installing a mod removes its MO2 placeholder

When a mod imported as a name-only MO2 placeholder is later installed
via Nexus, the original placeholder is removed. Two passes handle a
timing race: at install time (`addModFromPath`) and when the Nexus
title arrives (`onTitleFetched`).

For mods whose `meta.ini` was resolved on import, `handleNxmUrl`
detects the existing placeholder by URL and promotes it in-place so
it stays in the same position.

### Enter key opens the selected mod

Same as double-click. Previously only Delete had a binding.

### Removing mods offers to delete from disk

A confirmation now asks "you have N installed mod folder(s) on disk.
Delete them too?". No is the default. Multi-select asks once.
